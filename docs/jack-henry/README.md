# Jack Henry Pilot Package

Outreach and pilot collateral for taking **CodexDominion BankTrust OS** into a
Jack Henry partnership conversation. This is a **partnership conversation, not a
feature sprint** — the product is built and demoable.

**Live demo:** https://codexdominion-command-console.vercel.app/banking

## Positioning line

> **CodexDominion BankTrust OS is the AI governance operating system for community
> and regional banking — helping institutions govern AI-assisted lending, vendor
> risk, BSA/AML workflows, consumer complaints, audit evidence, and regulatory
> readiness.**

## What to emphasize for Jack Henry

- Community / regional banking focus (their ecosystem)
- AI governance + audit readiness
- Vendor & third-party risk
- Lending decision oversight
- FFIEC / OCC / FDIC / CFPB readiness
- Evidence generation
- Integration-contract architecture (plugs into existing systems)
- BankTrust OS as an **industry edition**, not a one-off app

## Honesty guardrails (keep claims accurate)

- We are **not** partnered with or integrated into Jack Henry today — outreach is
  to **explore fit**. Use conditional language ("could support institutions using
  Jack Henry ecosystems").
- The demo runs on **demonstration data**; live core-banking/LOS binding is
  available via the Integration Contract but not yet connected to a real bank.
- Don't claim certifications (SOC 2, FedRAMP). We help **produce the evidence**
  those programs require.
